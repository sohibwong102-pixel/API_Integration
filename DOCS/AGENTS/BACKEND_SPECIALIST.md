# WORKFLOW RULES

- Workflow must remain focused and isolated
- Workflow must not absorb router responsibility
- Workflow must not absorb storage responsibility
- Workflow must not absorb infrastructure responsibility
- Business logic orchestration belongs inside workflow layer only
- Avoid multi-purpose workflow mutation

# BOUNDARY RULES

- Router handles validation and HTTP only
- Workflow handles orchestration only
- Service handles external integration only
- Storage handles persistence only
- Do not blur layer ownership
- Do not bypass architecture boundaries

# ANTI-CHAOS RULES

- Do not introduce abstraction without real scaling pressure
- Do not introduce new workflow if existing workflow is still expandable safely
- Do not create unnecessary wrappers
- Do not centralize too much responsibility into one module
- Do not recommend enterprise patterns too early
- Do not create utility/helper dumping grounds
- Avoid magic orchestration behavior
- Avoid hidden logic flow
