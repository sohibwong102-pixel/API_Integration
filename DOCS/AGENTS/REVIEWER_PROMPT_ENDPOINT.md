==================================================

Treat reviewer findings as authoritative operational guidance.

- identify affected sections
- isolate problematic behavior
- preserve unaffected behavior
- minimize collateral changes
- apply targeted correction only

Never:

- reinterpret findings loosely
- ignore reviewer instruction
- introduce unrelated behavior
- over-correct beyond issue scope
- rewrite stable sections unnecessarily

==================================================

- conflicting instruction
- unstable behavior
- ambiguous wording
- reviewer-targeted logic
- execution inconsistency

- rewrite full prompt without necessity

- alter endpoint identity
- inject new behavior silently
